# Qahse 风场输入文件 (.qwd) 评估与改进建议

## 一、 对 IEC 61400 陆上与海上标准的覆盖情况

根据现有的 `.qwd` 文件参数结构，Qahse 的风场生成器 **基本可以覆盖 IEC 61400-1（陆上）和绝大部分 IEC 61400-3（海上）标准对风况的基本要求**。

### 1. 陆上风机 (IEC 61400-1)

* **支持度极高**：文件明确包含了常规湍流模型 (NTM)、极端湍流 (ETM)、极端风速 (EWM)、极端运行阵风 (EOG)、极端方向变化 (EDC)、极端相干阵风 (ECD) 和极端风切变 (EWS) 等多重风况定义。
* **参数联动**：支持 IEC 的标准版次选择 (`ED2/ED3/ED4`)，以及风力机等级 (`I/II/III`) 和湍流等级 (`A/B/C`) 的自动匹配。

### 2. 海上风机 (IEC 61400-3)

* **基本支持**：海上风机的“风”部分绝大多数直接继承自 IEC 61400-1 标准（如 Kaimal 谱），该文件能妥善处理这些常规需求。
* **潜在限制**：海上环境有时需要考虑非中性大气层结（Atmospheric Stability），或是针对特定热带海域的台风类（Class T）阵风模型。当前的参数列表中尚未见针对这些特殊气象的衍生项。

## 二、 与 TurbSim 和 Bladed 相比的改进与升级方向

相较于行业标杆工具（NREL 的 TurbSim 和 DNV 的 Bladed），Qahse 工具在算法效能（如 `USEFFT` 结合 MKL 加速，以及明确列出的 `SRM` 和经典 `VEERS` 算法）和直接集成度上具有现代化的特点。若要在功能和用户体验上进一步超越，建议考虑以下改进：

### 1. 物理模型与特定标准的增强

1. **支持非中性大气稳定性 (Atmospheric Stability)**：
   * *问题*：TurbSim 允许用户通过输入理查森数或莫宁-奥布霍夫长度模拟稳定或不稳定的大气边界层。对于海温与气温差异极大的海上风电，这一特性尤为重要。
   * *建议*：在 `.qwd` 中增加 `STABILITY_TYPE` 或 `RICHARDSON_NUM` 参数。
2. **增加低空急流 (Low-Level Jets, LLJ)**：
   * *问题*：在大叶轮直径的海上风机和现代高塔筒陆上风机中，低空急流对载荷的影响很大。
   * *建议*：引入专门的 LLJ 剖面高度与强度控制参数。
3. **支持台风抗台等级 (Class T)**：
   * *问题*：针对最新版标准（如 IEC 61400-1 Ed.4 和相关中国国标），需支持超强台风工况。
   * *建议*：修改 `TURBINECLASS` 的校验逻辑，支持 `T` 级别及其对应的极值风速算法。
4. **空间相干性 (Spatial Coherence) 调整**：
   * *问题*：Bladed 和 TurbSim 允许高级用户自定义空间相干谱的衰减系数，目前的配置可能是内部写死了 IEC 的默认拟合值。
   * *建议*：将其开放为高级可选参数，提升研发灵活性。

### 2. 文件格式、工具链接口与批处理体验改进

1. **输入文件格式现代化**：
   * *问题*：当前格式为 `VALUE KEYWORD - Description`，属于定制型正则解析格式，不够通用。TurbSim 的 `.inp` 是位置敏感的纯纯文本，而 Bladed 主要依赖重度 JSON/XML 或 GUI。
   * *建议*：采用标准且对人友好的 **YAML** 或 **TOML** 格式（例如：`MEANWINDSPEED: 11.4  # [m/s]`）。这将极大方便 Python 等脚本语言进行解析与参数修改。
2. **风场矩阵的批量生成 (Batch Matrix Generation)**：
   * *问题*：真实风机载荷计算（DLC 1.2 等）需要庞大的湍流风测试矩阵（如风速 $v_{in}$ 到 $v_{out}$，每个风速 6 个不同随机种子）。
   * *建议*：允许在配置中给定列表/范围输入（如 `TURBSEED: [1,2,3,4,5,6]`），让解析器一次性生成整个风速或种子范围的风场。
3. **测风塔实测时间序列的“混音”覆盖 (Time Series Mix-in)**：
   * *问题*：当 `MODE` 为 `IMPORT` 时，若只进行简单的时间插值，依然有数据缺陷。
   * *建议*：增强导入模式，允许将测风塔获得的真实非连续时间序列作为三维低频背景基准，并在此基础上通过 Qahse 的 Kaimal 算法叠加高频合成湍流。这一功能将使其远超 TurbSim 繁琐的外部数据合并流程。
